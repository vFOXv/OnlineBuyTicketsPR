Используемые технологиии:
Java, Spring Boot, Thymeleaf, HTML, JPA, Validation, Lombok, Spring Security.

Запуск приложения: в поискокой строке ввести localhost:8080

Приложение моделирует работу онлайн приложения по покупке билетов на рейсовые автобусы.

DB - код для создания и заполнения DB в папке resources/initDB.sql

Все водимые данные должны быть на латинской раскладке, а также цифры и символы.

При запуске по URL "/" открывается стартовая страница с функционалом:
- авторизация юзера
- создание нового юзера
- логаут

При создании нового юзера происходит проверка на наличее юзера с таким же именем,
совпадения пароля и его повторения, минимальная длина имени и пароля(4 символа), и валидность email.

После успешной регистрации происходит переход на страницу авторизации.

После авторизации юзер попадает в меню(роль USER):
- Расписание рейсов
- Список моих билетов
- Разлогинивание

- Расписание рейсов показывает куда, когда и сколько мест/свободных мест на рейсе(если свободных мест нет,
    то поле Buy ticket блокируеться и выделяеться красным цветом)
    - при переходе по ссылке 	"Buy ticket on the bus flights" юзер попадает в окно покупки билета,
    он выбирает номер места из списка, и вводит имя и фамилию пассажира.
    Имя и фамилия только латиницей, цифрами и символами. Валидация на длинну строки(минимум два символа).
    После успешной покупки юзера перебрасывает на страницу Расписание рейсов.

- Список моих билетов показывает списов билетов юзера, при переходе по ссылке "Show ticket",
    дается полное описание билета и возможность его сдать(кнопка DELETE).
    После сдачи билета юзура перебрасывает на страницу Расписание рейсов.

После авторизации юзер попадает в меню(роль ADMIN):
- Расписание рейсов
- Список моих билетов
- Вход в админ часть (ссылка видна только админу)
- Разлогинивание

При преходе в амин часть админ попадает в админ меню:
-Список всех билетов
-Список всех пользователей
-Создание нового рейса

-Список всех билетов - переход на страницу со списком всех билетов и возможность поиска по рейсам и юзерам
    и возможностью просмотра и удаления билетов(после удаления юзера перебрасывает на страницу со списком всех билетов)

-Список всех пользователей - переход на страницу со списком всех юзеров, возможность просмотреть информацию по выбранному юзеру
    и забанить юзера(поле active). Если юзер забанен, то после авторизации его сперва перекидывает на /logout
    а потом опять перебрасывает на стартовую страницу.

-Создание нового рейса - поля заполняються на латинице, цифрами и символами. Валидация на пустые поля и
    и поле amount seats на число сиденеий > 0.
